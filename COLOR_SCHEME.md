## üé® Color Palette Summary

### Tailwind Color Reference
Your theme is based on these Tailwind color scales:

**Light Mode:**
- **Background/Cards**: `zinc-50` to `white`
- **Text**: `zinc-900` (`#09090b`)
- **Borders**: `zinc-200` (`#e4e4e7`)
- **Muted Text**: `zinc-500` (`#71717a`)
- **Secondary BG**: `zinc-100` (`#f4f4f5`)
- **Accent**: `zinc-800` to `zinc-900` (`#18181b`)

**Dark Mode:**
- **Background/Cards**: `zinc-950` (`#09090b`)
- **Text**: `zinc-50` (`#fafafa`)
- **Borders**: `zinc-800` (`#27272a`)
- **Muted Text**: `zinc-400` (`#a1a1aa`)
- **Secondary BG**: `zinc-800` (`#27272a`)
- **Accent**: `zinc-50` (`#fafafa`)

### Accent Color: Red (Destructive Only)
- Light: `red-500` (`#ef4444`)
- Dark: `red-900` (`#7f1d1d`)

## üõ†Ô∏è Using the Color Scheme

### In Tailwind Classes
```tsx
// Background colors
<div className="bg-background">        // White (light) / Near-black (dark)
<div className="bg-card">              // Card background
<div className="bg-secondary">         // Light gray / Dark gray
<div className="bg-muted">             // Muted background

// Text colors
<p className="text-foreground">        // Primary text
<p className="text-muted-foreground">  // Secondary/subdued text
<p className="text-destructive">       // Error/delete text

// Border colors
<div className="border border-border"> // Standard border
<div className="ring ring-ring">       // Focus ring

// Button variants (from shadcn/ui)
<Button variant="default">             // Primary (dark bg, light text)
<Button variant="secondary">           // Secondary (light bg, dark text)
<Button variant="destructive">         // Delete/error (red)
<Button variant="ghost">               // Transparent with hover
<Button variant="outline">             // Bordered, transparent bg
```

### Custom CSS with Variables
```css
.custom-element {
  background-color: var(--background);
  color: var(--foreground);
  border: 1px solid var(--border);
}

.custom-card:hover {
  background-color: var(--accent);
  color: var(--accent-foreground);
}
```

## üéØ Design Principles

### 1. **Neutral & Professional**
- Monochromatic gray scale (zinc palette)
- No vibrant brand colors (except red for destructive actions)
- Clean, minimal aesthetic
- Similar to: Linear, Notion, Arc Browser

### 2. **High Contrast for Readability**
- Light mode: Near-black text on white (`#09090b` on `#ffffff`)
- Dark mode: Near-white text on near-black (`#fafafa` on `#09090b`)
- WCAG AAA compliant for body text

### 3. **Subtle Interactions**
- Light gray hover states (`#f4f4f5` light, `#27272a` dark)
- Focus rings for accessibility (`ring-ring`)
- Smooth transitions between states

### 4. **Consistent Spacing**
- Border radius: `0.5rem` (8px) - rounded corners
- Consistent padding/margins from Tailwind defaults

## üîÑ Theme Toggle Behavior

Your app uses a `.dark` class on the root element:

```tsx
// Light mode
<html class="">
  // Uses :root CSS variables
</html>

// Dark mode
<html class="dark">
  // Uses .dark CSS variables
</html>
```

Theme is persisted in localStorage as `'light'` or `'dark'`.

## üí° Customization Ideas

If you want to add a custom brand color later:

### Option 1: Add an Accent Color (e.g., Blue)
```css
:root {
  --primary: #3b82f6;  /* Blue-500 */
  --primary-foreground: #ffffff;
}

.dark {
  --primary: #60a5fa;  /* Blue-400 (lighter for dark mode) */
  --primary-foreground: #1e3a8a;  /* Blue-900 */
}
```

### Option 2: Add Brand Colors
```css
:root {
  --brand: #8b5cf6;           /* Purple-500 */
  --brand-foreground: #ffffff;
  --success: #22c55e;         /* Green-500 */
  --warning: #f59e0b;         /* Amber-500 */
  --info: #3b82f6;            /* Blue-500 */
}
```

Then use them:
```tsx
<Button className="bg-[var(--brand)] text-[var(--brand-foreground)]">
  Upgrade
</Button>
```
